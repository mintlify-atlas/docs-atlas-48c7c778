---
title: useCanvasEditor
description: Hook for managing canvas editing state with drawing, text, and image tools
---

The `useCanvasEditor` hook provides a complete canvas editing system with support for drawing paths, adding text, placing images, and managing layers.

## Usage

```typescript
import { useCanvasEditor } from "@/hooks/use-canvas-editor";

function CanvasScreen() {
  const {
    paths,
    texts,
    images,
    zOrder,
    activeTool,
    selectedColor,
    selectedStrokeWidth,
    setActiveTool,
    addPath,
    addText,
    addImage,
    undo,
    clear,
    getState,
    loadState,
  } = useCanvasEditor();

  // Draw a path
  const handleDrawComplete = (pathData: string) => {
    addPath(pathData, selectedColor, selectedStrokeWidth);
  };

  // Add text
  const handleAddText = () => {
    addText("Hello F1", selectedColor, 100, 100);
  };

  // Save state
  const handleSave = () => {
    const state = getState();
    saveToDatabase(state);
  };

  return (
    <Canvas
      paths={paths}
      texts={texts}
      images={images}
      zOrder={zOrder}
      tool={activeTool}
      onDrawComplete={handleDrawComplete}
    />
  );
}
```

## Return Values

### State

<ResponseField name="paths" type="DrawPath[]">
  Array of drawn paths on the canvas
  
  ```typescript
  type DrawPath = {
    id: string;
    path: string;  // SVG path data
    color: string;
    strokeWidth: number;
  };
  ```
</ResponseField>

<ResponseField name="texts" type="CanvasText[]">
  Array of text elements on the canvas
  
  ```typescript
  type CanvasText = {
    id: string;
    text: string;
    color: string;
    x: number;
    y: number;
    scale: number;  // Default: 1.5
  };
  ```
</ResponseField>

<ResponseField name="images" type="CanvasImage[]">
  Array of image elements on the canvas
  
  ```typescript
  type CanvasImage = {
    id: string;
    uri: string;
    x: number;
    y: number;
    width: number;
    height: number;
  };
  ```
</ResponseField>

<ResponseField name="zOrder" type="string[]">
  Array of element IDs in z-order (bottom to top)
</ResponseField>

<ResponseField name="undoStack" type="CanvasAction[]">
  Stack of actions for undo functionality
</ResponseField>

<ResponseField name="activeTool" type="ToolMode">
  Currently selected tool: `"draw"`, `"text"`, `"image"`, or `"eyedropper"`
</ResponseField>

<ResponseField name="selectedColor" type="string">
  Currently selected color for drawing/text
</ResponseField>

<ResponseField name="recentColors" type="string[]">
  Array of recently used colors (max 6)
</ResponseField>

<ResponseField name="selectedStrokeWidth" type="number">
  Currently selected stroke width for drawing (default: 3)
</ResponseField>

### Tool Functions

<ResponseField name="setActiveTool" type="(tool: ToolMode) => void">
  Set the active tool mode
</ResponseField>

<ResponseField name="setSelectedColor" type="(color: string) => void">
  Set the selected color and add it to recent colors
</ResponseField>

<ResponseField name="setSelectedStrokeWidth" type="(width: number) => void">
  Set the stroke width for drawing
</ResponseField>

<ResponseField name="updateText" type="(id: string, updates: Partial<Pick<CanvasText, 'x' | 'y' | 'scale'>>) => void">
  Update position or scale of a text element
</ResponseField>

<ResponseField name="updateImage" type="(id: string, updates: Partial<Pick<CanvasImage, 'x' | 'y' | 'width' | 'height'>>) => void">

### History Operations

<ResponseField name="undo" type="() => void">
  Undo the last action (removes the most recently added element)
</ResponseField>

<ResponseField name="clear" type="() => void">
  Clear the canvas and reset all state to defaults
</ResponseField>

<ResponseField name="getState" type="() => CanvasState">

    images: CanvasImage[];

## Tool Modes

The canvas supports four tool modes:

- `"draw"` - Drawing paths with selected color and stroke width
- `"text"` - Adding text elements
- `"image"` - Placing images on the canvas
- `"eyedropper"` - Selecting colors from the canvas

## Implementation Details

- Each element (path, text, image) is assigned a unique ID using timestamp and counter
- Z-order is managed separately from element arrays for efficient layer control
- Recent colors are limited to 6 entries
- Undo removes the last added element and updates z-order accordingly
- State persistence includes all elements and current tool settings
