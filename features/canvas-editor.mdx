The canvas editor provides a full-featured drawing interface for customizing bingo cells. Users can draw freehand, add text, insert images, and use advanced tools like color picking and layering.

## Tool modes

The editor supports five tool modes defined in `canvas-types.ts`:

```typescript canvas-types.ts
export type ToolMode = "draw" | "text" | "image" | "hand" | "pipette" | null;
```

### Recent colors

The editor automatically tracks recently used colors:

```typescript use-canvas-editor.ts
const setSelectedColor = useCallback((color: string) => {
  setSelectedColorState(color);
  setRecentColors((prev) => {
    if (prev.includes(color)) return prev;
    const newColors = [color, ...prev];
    if (newColors.length > 6) newColors.pop();
    return newColors;
  });
}, []);
```

## Eyedropper tool

The pipette/eyedropper tool captures the canvas and allows color picking:

```tsx canvas-editor-modal.tsx
const handleToolChange = useCallback(
  async (tool: ToolMode) => {
    if (tool === "pipette") {
      if (viewShotRef.current) {
        try {
          const uri = await captureRef(viewShotRef.current, {
            format: "png",
            quality: 1,
            result: "base64",
          });
          setEyedropperUri(uri);
        } catch (e) {}
      }
    }
    editor.setActiveTool(tool);
  },
  [editor]
);
```

When activated, the `EyedropperLayer` component overlays the canvas with a touch-interactive color picker.

<Info>
The eyedropper uses ViewShot to capture a base64 snapshot of the canvas, which is then analyzed for color picking.
</Info>

## Drag and drop

Text and image elements can be dragged using React Native Gesture Handler:

### Drag to delete

When dragging, a trash zone appears at the bottom of the screen. Dropping an element on the trash deletes it:

```tsx canvas-editor-modal.tsx
const handleDragEnd = useCallback(
  (id: string, type: "text" | "image", pageY: number) => {
    setIsDragging(false);
    const { y, height } = trashLayoutRef.current;
    if (y > 0 && pageY >= y && pageY <= y + height) {
      if (type === "text") {
        editor.removeText(id);
      } else {
        editor.removeImage(id);
      }
    }
  },
  [editor]
);
```

### Z-order management

When a drag starts, the element is automatically brought to front:

```typescript use-canvas-editor.ts
const bringToFront = useCallback((id: string) => {
  setZOrder((prev) => {
    const arr = prev.filter((item) => item !== id);
    arr.push(id);
    return arr;
  });
}, []);
```

## Undo functionality

The editor maintains an undo stack that tracks all added elements:

```typescript use-canvas-editor.ts
const undo = useCallback(() => {
  setUndoStack((prev) => {
    if (prev.length === 0) return prev;
    const last = prev[prev.length - 1];
    const newStack = prev.slice(0, -1);

    switch (last.type) {
      case "draw":
        setPaths((p) => p.filter((item) => item.id !== last.id));
        break;
      case "text":
        setTexts((t) => t.filter((item) => item.id !== last.id));
        break;
      case "image":
        setImages((i) => i.filter((item) => item.id !== last.id));
        break;
    }
    setZOrder((z) => z.filter((id) => id !== last.id));
    return newStack;
  });
}, []);
```

<Warning>
Undo only removes the most recently added element. Moving or modifying existing elements does not add to the undo stack.
</Warning>

## Saving and loading

The canvas state can be saved and restored:

```typescript use-canvas-editor.ts
const getState = useCallback((): CanvasState => {
  return { 
    paths, 
    texts, 
    images, 
    zOrder, 
    selectedColor, 
    selectedStrokeWidth, 
    recentColors 
  };
}, [paths, texts, images, zOrder, selectedColor, selectedStrokeWidth, recentColors]);

const loadState = useCallback((state: CanvasState | null) => {
  if (state) {
    setPaths(state.paths);
    setTexts(state.texts);
    setImages(state.images);
    setZOrder(state.zOrder || []);
    setSelectedColorState(state?.selectedColor ?? CANVAS_COLORS[0]);
    setRecentColors(state?.recentColors ?? [...CANVAS_COLORS]);
  }
}, []);
```

When saving, the canvas is captured using ViewShot:

```tsx canvas-editor-modal.tsx
const handleSave = useCallback(async () => {
  try {
    if (viewShotRef.current?.capture) {
      const uri = await viewShotRef.current.capture();
      const state = editor.getState();
      onSave(uri, state);
      editor.clear();
    }
  } catch {
    Alert.alert("Error", "Failed to save the canvas");
  }
}, [onSave, editor]);
```
